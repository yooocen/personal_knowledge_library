```shell
dd if=/dev/zero of=ext2.img bs=1M count=100
```


这条命令创建一个 100MB 的空文件 ext2.img：
- dd — 数据复制工具（disk duplicator）
- if=/dev/zero — 输入文件是 /dev/zero，会无限输出 \0 字节
- of=ext2.img — 输出文件
- bs=1M — 每次读写 1MB（块大小）
- count=100 — 读写 100 次
结果是：一个全零填充的 100MB 文件，通常用来做 loop 设备挂载——后面你可以 mkfs.ext2 ext2.img 格式化成 ext2 文件系统，然后 mount -o loop ext2.img /mnt 挂载使用

```shell
mkfs.ext2 -r 0 ext2.img
```

创建 ext2 文件系统时指定修订版本 (revision level) 为 0。
- -r 0 表示使用旧版 ext2 格式（revision 0，也称为 ext2 "good old" 格式），而不是默认的 revision 1（动态文件系统）
- revision 0 的限制：inode 大小固定为 128 字节，不支持扩展属性 (xattr)、ACL、大文件（超过 2TiB）等特性

```
dumpe2fs ext2.img
```

显示 ext2.img 文件系统的超级块和块组信息。
输出内容包括：inode 数量、块数量、块大小、修订版本、挂载计数、最后挂载时间、块组信息等。通常用于调试或查看文件系统创建参数。
常用选项：
- -h — 只显示超级块信息，不列出块组
- -b — 打印块组中保留块的详细信息

```
debugfs -w ext2.img
```

以读写模式打开 ext2.img，进入 ext2/3/4 文件系统的交互式调试 shell。
