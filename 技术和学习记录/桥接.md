# docker中的桥接
**Bridge（桥接）​ 的本质是：**
- Docker 在宿主机上创建一个 **虚拟交换机（docker0 或自定义网桥）**
- 每个容器连接到这个虚拟交换机上
- 容器之间可以通过这个交换机通信
- 通过 NAT 技术访问外网

网桥 == 交换机

# 光猫的桥接模式
光猫不负责拨号，路由器进行拨号，光猫只负责光信号的中转


为什么名字都叫“桥接”
因为它们都来自同一个网络概念：
> **Bridge（桥） = 把多个网络连在一起**
但实现方式完全不同：
- 路由器桥接：**二层桥**
- Docker bridge：**二层 + 三层混合桥**
