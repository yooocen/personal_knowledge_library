## 1. 意图框架在做什么
- 将“用户想做什么”抽象为系统级标准（意图），让 HarmonyOS 能在合适的时间/入口预测、推荐或触发业务能力。
- 一般链路可以理解为：开发者描述/共享业务意图 -> 系统学习用户偏好与场景特征 -> 在系统入口/小艺入口/应用内入口分发 -> 用户点击或语音触发 -> 系统调用意图落到具体服务页/能力执行。

## 2. 用户侧入口与场景（示例页）
### 2.2.2 智慧搜索
- 定位：系统提供的快速搜索工具，可搜索本机内容（应用/联系人/设置/备忘录等）以及网络内容。
- 进入方式：从主屏幕中部下滑打开智慧搜索（文档示例页）。
- 使用方式要点：
  - 在输入框输入应用/文件名快速定位。
  - 可将应用固定到搜索界面“快捷位置”。
  - 可查看搜索历史、管理搜索范围。
  - 预置应用可删除（在搜索列表中长按删除）。

### 小艺相关交互入口（示例页）
- 语音唤醒/快捷操作：包含“按电源键 1 秒唤醒小艺”、或配置“语音唤醒词”等方式（示例页给出设置路径提示）。
- 小艺建议：强调基于时空信息与推荐算法进行服务推荐，并覆盖多设备/多入口。

## 3. 推荐类场景的运行过程（示例：视频意图）
系统智能推荐意图的典型步骤（示例页）：
1. 开发者向 HarmonyOS 进行“播放视频”等意图共享（完成时机由业务决定）。
2. 系统基于共享意图与用户感知做多维特征学习（例如时间、位置、设备状态、运动状态等）。
3. 系统在合适时机于小艺建议推荐对应意图卡片。
4. 用户点击卡片后，系统发起意图调用，拉起对应应用/页面完成播放等任务。

补充：文档提到在某些推荐场景下会有“辅助推荐加持（将来时模式）”等特殊情况，用于提升体验（示例页仅展示了开头说明）。

## 4. 生态接入（2.5）
### 2.5.1 接入范围
- 意图框架支持 HarmonyOS 应用与元服务等载体进行对接。
- 对部分意图的“意图调用/意图共享”，可能需要业务自有云侧后台与华为侧服务进行对接（文档示例页表述为需“云侧后台与华为侧服务端对接”）。
- 具体支持清单与范围：文档提示见“附录 1/附录 2”（本总结仅覆盖已看到的附录页）。

### 2.5.2 接入方式（概念）
- 智慧分发场景通常由多个意图组合为“智慧服务”。
- 接入时选择合适的智慧服务，并按不同智慧场景的必选条目完成对接，实现多种智慧场景分发。

### 2.5.3 接入流程（四阶段）
开发者接入整体分为 4 个阶段（示例图）：
- 准备：先阅读并理解意图框架规范，结合自身业务选取合适的智慧分发场景与对应智慧服务/意图。
- 开发：云侧对接华为服务端，端侧开发意图共享能力（示例图中体现“云：对接华为服务端”“端：开发意图共享”）。
- 注册：应用/元服务上架并注册意图（示例图强调“上架并注册意图”）。
- 分发：通过 HarmonyOS 系统入口进行意图服务分发。

## 5. 技术术语（示例页）
- `HarmonyOS意图框架`：HarmonyOS 提供的一套系统级意图标准体系。
- `意图`：开发者与 HarmonyOS 对接的标准元素，一个意图代表应用/元服务对用户提供的某个业务功能。
- `元服务`：基于 HarmonyOS API 的新服务提供方式，一次开发可部署在 1+8+N 设备上，可通过卡片等轻量形态提供服务。

## 6. 意图共享（3.4，示例页）
### 6.1 核心目的
- 开发者通过意图共享“协助系统学习用户意图规律”，并在合适时机于系统入口进行推荐。
- 与意图调用类似，意图共享也提供端侧和云侧两种对接方式（示例页明确提到“两种意图共享对接方式”）。

### 6.2 端侧意图共享接口（示例页列出的 3 个）
文档示例页展示端侧包含 3 个功能接口（接口名以示例页为准）：
- `shareIntent(InsightIntent)`：共享意图（支持多个意图批量新增），共享意图已发生的“概要发生”的描述。
- `deleteIntent(InsightIntent)`：删除共享意图（按意图名和意图唯一标识符等删除）。
- `deleteEntity(InsightIntent)`：删除实体（按实体标识 `entityId` 删除某实体；删除后相关实体的搜索与推荐能力将不再支持）。

并展示了 `InsightIntent` 数据结构示例（页中可见字段例如 `identifier` 为共享意图唯一 ID，类型 `string`，必填）。

## 8. 接入落地建议（基于文档已覆盖内容）
- 先选定目标入口与场景：例如优先做“系统推荐卡片”或“智慧搜索可检索/直达”。
- 明确你要做的是：
  - `意图共享`：让系统能学习并推荐你；
  - `意图调用`：让系统能把用户动作落到你业务页面/能力。
- 按“准备-开发-注册-分发”四阶段推进，尤其注意云侧对接诉求与端侧接口实现的边界。
