fork的话可以共享 struct file ^LBWP2Ie2

为什么需要这一层？
两个进程打开同一个文件，各自有独立的 struct file：
// 进程 A
int fd1 = open("/tmp/log", O_WRONLY);  // 创建 struct file X, f_pos=0
write(fd1, "AAA", 3);                  // X.f_pos → 3

// 进程 B
int fd2 = open("/tmp/log", O_WRONLY);  // 创建 struct file Y, f_pos=0
write(fd2, "BBB", 3);                  // Y.f_pos → 3
// 结果：文件内容是 "AAABBB"（各自独立偏移，不覆盖）

如果没有 struct file 这一层，f_pos 就得存在 inode 里，两个进程会互相覆盖对方的写入位置，无法并发安全地操作文件。 ^QggrvssT
