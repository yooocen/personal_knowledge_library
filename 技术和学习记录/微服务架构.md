# CAP原则
C consistency
A abilitiablity
P partition tolerance
三者不可兼得，像银行这种结账不可以出现不一致的系统，c是一定要保障的，A和P就只能二选一

# 负载均衡
- 集中式负载均衡（服务器负载均衡），在消费者和生产者之间使用独立的负载均衡设施，比如说Nginx
- 进程内负载均衡（客户端负载均衡），将负载均衡集成到消费者，消费者从服务注册中心获知哪些服务地址可用，然后自己从这些地址中选取一个可用的生产者，比如Ribbon
