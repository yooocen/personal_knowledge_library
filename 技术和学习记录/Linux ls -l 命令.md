在Linux系统中，`ll`命令通常是`ls -l`的别名，用于列出目录中文件的详细信息。`ll`或`ls -l`打印的每一行包含以下信息：

1. **文件类型和权限**: 第一个字符表示文件的类型，比如 `-` 表示普通文件，`d` 表示目录，`l` 表示符号链接等。接下来的九个字符表示文件的权限，分为三组，每组三个字符分别表示文件所有者、所属组和其他用户的读、写和执行权限。
2. **连接数（Links）**: 这个数字表示指向该文件的硬链接数量。对于目录，这个数字通常是 2 加上该目录的子目录数量，因为每个目录都有两个硬链接，一个是指向自身的 `.`，另一个是指向父目录的 `..`。
3. **所有者（Owner）**: 文件或目录的所有者用户名。
4. **所属组（Group）**: 文件或目录所属的组名。
5. **文件大小（Size）**: 文件的大小，以字节为单位。对于目录和符号链接，这个值可能不准确。
6. **修改时间或最近访问时间（Modification time）**: 文件内容最后一次修改的时间。对于符号链接，表示链接的目标文件最后一次修改的时间。

7. **文件名（File name）**: 文件或目录的名称。

例如，以下是`ll`或`ls -l`命令输出的一个示例行：

```
-rw-r--r-- 1 user group 4096 Jan  1 00:00 example.txt
```

在这个示例中，每一段的意义是：

- `-rw-r--r--`: 文件类型和权限。`-` 表示普通文件，`rw-r--r--` 表示文件所有者有读写权限，所属组和其他用户有只读权限。
- `1`: 连接数，表示有一个硬链接指向该文件。
- `user`: 文件的所有者用户名。
- `group`: 文件所属的组名。
- `4096`: 文件大小，以字节为单位。
- `Jan  1 00:00`: 文件的修改时间。
- `example.txt`: 文件名。

在Linux系统中，常见的文件类型包括：

1. **普通文件（Regular file）**：包含数据的常规文件。可以是文本文件、二进制文件等。在`ls -l`输出中，普通文件的类型用 `-` 表示。
2. **目录（Directory）**：包含其他文件或目录的文件。在`ls -l`输出中，目录的类型用 `d` 表示。
3. **符号链接（Symbolic link，也称软链接）**：指向另一个文件的特殊文件。在`ls -l`输出中，符号链接的类型用 `l` 表示。
4. **设备文件（Device file）**：用于与硬件设备通信的特殊文件。包括块设备文件和字符设备文件两种类型。块设备文件用于访问块设备（如硬盘），字符设备文件用于访问字符设备（如键盘、串口等）。在`ls -l`输出中，块设备文件的类型用 `b` 表示，字符设备文件的类型用 `c` 表示。
5. **管道文件（Named pipe，也称FIFO）**：用于进程间通信的特殊文件。在`ls -l`输出中，管道文件的类型用 `p` 表示。
6. **套接字（Socket）**：用于进程间通信的特殊文件，主要用于网络通信。在`ls -l`输出中，套接字的类型用 `s` 表示。

这些文件类型可以通过`ls -l`或`ll`命令输出的第一个字符来区分。
