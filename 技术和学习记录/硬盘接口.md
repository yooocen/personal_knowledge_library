# 接口

消费级：
- SATA（Serial ATA）：最常见家用盘接口，6Gbps，AHCI 协议
- M.2 SATA：物理形态 M.2 但走 SATA 总线（和下面 NVMe 外观一样但不同协议）
- M.2 NVMe：走 PCIe 总线，协议 NVMe，高带宽低延迟（当前主流）
企业级：
- SAS（Serial Attached SCSI）：SCSI 协议的串行版，双端口、支持多路连接，比 SATA 贵但可靠
- U.2 / U.3：企业级 NVMe SSD 的物理接口（2.5寸），走 PCIe
- EDSFF（E3/E1）：新企业标准，散热和密度优化
数据中心互联：
- NVMe-oF（NVMe over Fabrics）：通过 RDMA/FC/TCP 把 NVMe 协议延伸到网络
- FC（Fibre Channel）：传统 SAN 网络
- iSCSI：IP 网络上的 SCSI
嵌入式/移动：
- eMMC：嵌入式 MMC，手机/平板内置存储
- UFS（Universal Flash Storage）：eMMC 的继任者，全双工，速度快
历史淘汰：
- PATA / IDE：并口排线
- SCSI（并行）：被 SAS 取代

## 补充

**IDE / PATA**（Parallel ATA）
- 并行排线（40/80针），16 bit 数据线
- 一个通道最多挂 2 设备（Master/Slave）
- 已被 SATA 完全淘汰

**USB**
- **USB Mass Storage (UMS)**：将盘模拟为 SCSI 设备，通过 USB 传输 SCSI 命令（Bulk-Only Transport，UAS 协议）
- **UAS**（USB Attached SCSI）：UMS 的升级版，支持 NCQ 和多队列，延迟更低
- USB 3.x 理论 5-20 Gbps，但协议开销和转换桥延迟是瓶颈

IDE = ATA 的并行版本，AHCI 是它的串行继承者。
USB = 外接场景的传输管道，封装的还是 SCSI/NVMe 协议。

# 协议

按层次从低到高：
## 物理传输协议（怎么连）

- **SATA**：串行 ATA，家用主流，命令集 AHCI
- **SAS**：串行 SCSI，企业级，双端口高可靠
- **PCIe**：NVMe SSD 走的物理总线
- **FC**：光纤通道，SAN 网络物理层
- **Ethernet**：iSCSI/NVMe-oF TCP 的底层

## 设备接口协议（怎么发命令）

- **AHCI**（Advanced Host Controller Interface）：SATA 盘的标准，一个队列，寄存器密集型
- **NVMe**：PCIe SSD 的原生协议，多队列无锁
- **SCSI**：SAS/UASP 的底层命令集（通过 SAM 模型抽像）
- **SATA Express / U.2**：物理形态，实际走 PCIe + NVMe 或 SAS

## 网络存储协议（怎么通过网络访问）

| 协议 | 传输载体 | 特点 |
|------|----------|------|
| **iSCSI** | TCP/IP | SCSI 命令封在 TCP 里走以太网 |
| **FC** | Fibre Channel | 专线 SAN，低延迟 |
| **FCoE** | 以太网 | FC 封装在以太网帧中 |
| **NVMe-oF** | RDMA/FC/TCP | 把 NVMe 队列延伸到网络，延迟最低 |
| **NFS** / **SMB** | IP 网络 | 文件级协议，不是块级 |
| **RBD (Ceph)** | TCP/IP | Ceph 分布式存储的块协议 |
| **iSER** | RDMA | iSCSI over RDMA |

## 软件协议 / 内核抽象

- **块层 bio/request**：内核内部统一的块 I/O 抽象
- **MD (multiple device)**：软 RAID
- **DM (device mapper)**：LVM / dm-crypt / dm-cache 的底层
- **loop**：文件作为块设备
- **nbd**：网络块设备

关键的演进方向：**SCSI → AHCI → NVMe**，网络方向：**FC → iSCSI → NVMe-oF**。
