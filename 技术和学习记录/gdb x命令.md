在 **GDB**​ 里，`x`是 **examine（检查内存）**​  的命令，用来查看任意地址的内存内容。这是逆向、调试、内核、嵌入式开发中最常用的命令之一。

---

## 一、`x`命令的基本语法

```
x /nfu addr
```

|部分|含义|
|---|---|
|`x`|examine|
|`n`|显示 **多少个单位**​|
|`f`|**显示格式**​|
|`u`|**单位大小**​|
|`addr`|要查看的内存地址|

> 顺序固定：`x /nfu addr`

---

## 二、常用格式说明（`f`）

|格式|含义|
|---|---|
|`x`|十六进制（hex）✅|
|`d`|十进制|
|`u`|无符号十进制|
|`o`|八进制|
|`t`|二进制|
|`c`|字符|
|`s`|字符串|
|`i`|汇编指令|

---

## 三、单位大小说明（`u`）

|单位|含义|
|---|---|
|`b`|byte（1 字节）|
|`h`|halfword（2 字节）|
|`w`|word（4 字节）|
|`g`|giant（8 字节）|

---

## 四、常见示例（非常重要）

### 🔹 1. 查看某个地址的 16 个字节

```
x /16xb 0x7fffffffe000
```

👉 16 个字节，十六进制

---

### 🔹 2. 查看指针指向的内容

```
x /4xw ptr
```

👉 从 `ptr`开始，看 4 个 32 位字

---

### 🔹 3. 查看字符串

```
x /s str
```

---

### 🔹 4. 反汇编内存（看指令）

```
x /10i main
```

👉 从 `main`开始显示 10 条汇编指令

（等价于 `disassemble`）

---

### 🔹 5. 查看栈内存

```
x /20xw $rsp
```

（x86‑64）

```
x /20xw $sp
```

（ARM）

---

### 🔹 6. 结合变量使用

```
x /4dw &my_var
```

---

## 五、快捷用法

|命令|等价|
|---|---|
|`x addr`|`x /1xw addr`|
|`x/s addr`|显示 C 字符串|
|`x/10i addr`|反汇编|

---

## 六、实用技巧

### ✅ 自动递增地址

`x`之后继续按 **Enter**，会接着上次地址继续查看：

```
x /16xb buf
<Enter>
<Enter>
```

---

### ✅ 查看结构体

```
x /8xw &struct_var
```

---

### ✅ 查看函数前几条指令

```
x /5i func
```

---

## 七、常见坑

❌ 地址未映射 → segfault

✅ 确保地址属于当前进程空间

❌ 忘了单位大小

✅ 推荐始终显式写 `b/w/g`

---

## 八、速查表（背下来）

```
x /nfu addr
n = 数量
f = x d u o t c s i
u = b h w g
```
